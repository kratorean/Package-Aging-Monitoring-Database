Package-Aging Monitoring Database
Project Overview
This project was developed to improve operational visibility into aging packages within UPS NG Northern hubs by identifying shipments that arrived each facility but had not yet been scanned into dispatch workflows.
The system leverages PostgreSQL and SQL-based analytics to support near real-time operational monitoring and package-aging analysis.
________________________________________
Business Problem
Operational teams experienced limited visibility into packages that:
•	arrived at facilities, 
•	remained unassigned, 
•	or were not scanned into inbound delivery workflows. 
This created challenges involving:
•	delayed package movement, 
•	operational bottlenecks, 
•	limited accountability, 
•	and reduced visibility into aging packages. 
________________________________________
Project Objective
The objective of this project was to:
•	track inbound package activity, 
•	identify packages missing dispatch scans, 
•	calculate operational aging, 
•	and improve package visibility through structured database monitoring.

Methodology
The system integrates operational package activity datasets including:
•	inbound arrival reports, 
•	driver load reports, 
•	delivery and pickup manifest data. 
SQL-based logic was developed to:
1.	Identify unmatched package records 
2.	Detect missing dispatch scans 
3.	Calculate package aging durations 
4.	Generate operational visibility views 
5.	Support near real-time monitoring workflows 
The database structure was designed to support scalable operational analytics and reporting.
